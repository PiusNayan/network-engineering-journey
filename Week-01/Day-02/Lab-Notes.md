# Week 01 • Day 02 Lab Notes

## Lab Focus

Today’s lab focused on identifying and understanding the main components that make up a computer network. The activity was designed to connect the theory of network components to a simple practical setup using Cisco Packet Tracer.

## Objective

The goal of the lab was to:
- Recognize the role of end devices and intermediary devices
- Observe how devices are connected through transmission media
- Understand how data moves between devices in a basic network
- Practice basic troubleshooting and connectivity reasoning

## Lab Setup

A simple network was created with:
- End devices such as PCs
- An intermediary device such as a switch
- A connection medium such as Ethernet cable

The Packet Tracer file used for this activity is:
- [Week-01-Day-02-Small-Office-LAN.pkt](Packet-Tracer/Week-01-Day-02-Small-Office-LAN.pkt)

The setup helped show how a basic network is formed and how communication begins between devices.

## Key Observations

1. End devices are the devices that create, send, or receive data.
2. Intermediary devices help move data between end devices.
3. Physical connectivity is necessary before communication can occur.
4. Devices must also have logical configuration, such as valid IP addressing, to communicate successfully.
5. A network is not just one device; it is a combination of hardware, media, and rules that allow communication.

## Practical Notes

- A switch is important for connecting multiple devices in a local network.
- A cable or wireless connection is the path through which data travels.
- Communication depends on both the physical connection and the correct logical setup.
- If one device is not properly configured, communication may fail even if the cables are connected correctly.

## Troubleshooting Thinking

When checking connectivity, I considered:
- Whether the devices were physically connected
- Whether the correct cables were used
- Whether the devices were configured with matching network settings
- Whether the network components were functioning as expected

## Reflection on the Lab

This lab helped me understand that networking is more than just connecting devices. It involves planning, configuration, and understanding the purpose of each component. I now see that even a simple network depends on several elements working together correctly.

## Key Takeaway

A successful network depends on the correct combination of:
- End devices
- Intermediary devices
- Transmission media
- Logical configuration
- Proper communication rules
